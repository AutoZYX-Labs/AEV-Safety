# AEV Safety 标准化推进路线

[English](standardization-roadmap-en.md)

## 定位

本路线面向自主工程车辆、移动作业机器系统和移动物理 AI 的标准化讨论。它不是现行标准要求，也不是对任何标准组织的正式提案。它把 AEV Safety 的研究框架拆成可进入团体标准、行业标准、国家标准或企业技术规范的模块。

## 标准化判断

工程车辆智能化安全的标准化问题，不是从零建立一套完全独立的标准体系。更可行的路径是把已有标准连接起来：

- 用工程机械标准处理机器本体、工作装置和安全相关控制。
- 用道路车辆 SOTIF 和场景测试方法处理功能不足、触发条件和场景覆盖。
- 用 AI 安全方法处理数据、模型、AI 输出不足和运行时监控。
- 用矿山和施工安全指南处理运行区域、组织措施、远程监督和变更管理。
- 用机器人和移动物理 AI 标准化信号处理人机共享空间、物理交互和自主决策。

## 候选标准模块

| 模块 | 建议内容 | 主要输出 |
|---|---|---|
| 1. 范围与对象 | 明确对象是 MWMS，而不是单台车或单个 AI 模型 | 适用机器、场地、任务、排除项 |
| 2. 术语 | MWMS、WTDC、DMWT、作业后果、监督模式、恢复能力 | 中英文术语表 |
| 3. 场地与任务声明 | 地形、物料、工具、人员区、禁入区、通信、定位、监督 | WTDC 声明模板 |
| 4. 动态作业任务 | 移动、工作装置、物料交互、任务质量和恢复动作 | DMWT 任务模板 |
| 5. SOTIF 风险链 | 功能不足、触发条件、危险事件、作业后果、缓解措施 | 风险链表和场景库 |
| 6. 测试评价 | 仿真、封闭场地、实车、故障注入、混合人机作业 | 场景矩阵和通过准则 |
| 7. 运行监控 | 近失事件、接管、降级、恢复、任务质量偏差 | 运行数据字段和事件分类 |
| 8. Safety case | claims、arguments、evidence、assumptions、limitations | 安全案例模板 |

## 术语候选

| 缩写 | 中文 | English | 说明 |
|---|---|---|---|
| MWMS | 移动作业机器系统 | Mobile Working Machine System | 由车辆、工作装置、工具、物料、场地、监督和组织共同构成的安全论证对象 |
| WTDC | 作业场地与任务设计条件 | Worksite and Task Design Conditions | 对 ODD 的扩展，用于声明工程作业边界 |
| DMWT | 动态移动与作业任务 | Dynamic Moving and Working Task | 对 DDT 的扩展，用于描述移动、作业和恢复闭环 |
| Task consequence | 作业后果 | Task consequence | 覆盖人员、资产、质量、生产、环境和恢复 |
| Runtime Guardian | 运行时安全守护 | Runtime Guardian | 对 AI、控制和任务质量进行运行时约束和降级 |

## 可优先形成的条款骨架

### 1. WTDC 声明

每个自主工程车辆系统在测试和部署前，应声明：

- 场地类型、地图边界、禁入区和人员活动区。
- 地形、坡度、承载、湿滑、粉尘、遮挡和能见度。
- 物料类型、粒度、含水率、堆形、可铲掘性或可处理性。
- 工具、附件、工作装置和任务模式。
- 通信、定位、调度、急停、围挡、警示和远程监督条件。
- 不允许运行的条件和退出条件。

### 2. DMWT 分解

每个任务应拆成移动动作、工作装置动作、物料交互、质量判断和恢复动作。例如装载机铲装卸料循环可拆为接近、识别料堆、选择铲掘点、铲掘、满载率判断、倒车转向、对准卸料对象、卸料、返回、恢复。

### 3. SOTIF 风险链

每个安全相关场景应记录：

```text
WTDC / DMWT
  -> 功能不足
  -> 触发条件
  -> 危险事件
  -> 作业后果
  -> 缓解措施
  -> 证据
```

### 4. 运行监控

运行监控不应只记录碰撞和急停，还应记录：

- 低置信度、OOD、定位退化、通信退化。
- 路径跟踪误差、铲斗/工具姿态误差、任务质量偏差。
- 远程监督请求、响应时间、确认结果和误确认。
- 最小风险状态执行、恢复路径、现场处置和复测结果。

## 分阶段推进

| 阶段 | 目标 | 产物 |
|---|---|---|
| Phase 1 | 统一语言和范围 | 术语表、WTDC/DMWT 模板、初步 taxonomy |
| Phase 2 | 建立测试评价结构 | 场景矩阵、指标族、失败样本记录表 |
| Phase 3 | 构建标准建议稿 | 标准目录、条款骨架、引用标准映射 |
| Phase 4 | 企业和现场验证 | 匿名化案例、运行日志字段、safety case 样例 |
| Phase 5 | 公开标准化推进 | 团体标准或行业标准草案、研讨会材料、公开征求意见稿 |

## 证据边界

截至 2026-06-05，公开仓库仅把标准状态页、公开标准目录、官方项目页和公开企业页面作为状态和范围证据。条款级解释、符合性判断和认证判断必须回到授权标准文本、正式测试报告和现场运行证据。
